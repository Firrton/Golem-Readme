# Contributing

This repository holds the Golem Community Builder Programme description and the docs builders read before applying. Improvements that make the programme clearer to incoming builders are welcome — typo fixes, translation corrections, broken-link fixes, and clarifications to copy that lands cold.

## Reporting an issue or suggesting a change

Open a GitHub issue describing what's unclear or outdated, with a link to the file and line where relevant. For trivial fixes — typos, broken links, formatting nits — a PR straight away is fine; no need to open an issue first.

## Translation contributions

The repo runs in three languages: `en/`, `es/`, `pt/`. The three should stay in sync structurally — same sections, same deliverables, same links.

If you change one language, please mirror the change in the other two when you can. If you can only provide one language, open the PR anyway and flag it; the team will mirror the change.

Language conventions:

- `en/` — British English (decentralised, organisation, licence, programme).
- `es/` — neutral Spanish. Avoid Spain-only or Latin-America-only idioms. No "vosotros".
- `pt/` — neutral Portuguese, post-1990 acordo ortográfico.

## Voice and language rules

- Plain, builder-to-builder. No hype words (revolutionary, game-changing, seamless, unleash, empower, next-generation, AI-native).
- `$GLM` with the dollar sign, every time.
- `Golem Network` on first mention; `Golem` after.
- Be concrete — numbers, examples, real timing — over adjectives.

The [Builders Guide](./en/docs/builders_guide.md) is the canonical example of the voice. Match it.

## Applying to the programme

This file is about *documentation* PRs. If you want to apply to the programme as a builder, use the official form at <https://forms.golem.network/golem-builders-programme> — not a PR.
